About the Project

Code, Data and Usability

The MyMiTo (MYcenaean & MInoan TOmbs) Project aims at digitising and formalising funerary evidence from Mycenaean and Minoan contexts of the Late Bronze Age. It provides an accessible and practical tool for navigating, querying, and analysing archaeological data.

A secondary goal is to produce a reusable, open-source tool for managing legacy datasets of funerary contexts. It enables not only data visualisation and exploration, but also statistical, quantitative, and spatial analysis in an immediate and intuitive way, generating human-readable outputs. The project is currently developed independently by the author.

Affinity Explanation

Conceptual Explanation

Each archaeological context (e.g., a tomb) is described by a combination of intrinsic parameters (such as tomb morphology or burial type) and object-derived parameters (such as material, function, or decorative theme). These parameters may result from objective observations or interpretative analyses.

The algorithm calculates a value of affinity (or correlation) between contexts based on all available parameters. Initially, every parameter is treated with equal weight, but its contribution is dynamically adjusted according to:

  • its statistical rarity (automatically calculated)
  • its presence in a specific combination of three parameters (called a "triple")
  • optional user-defined weighting or exclusion

Object-to-Object Comparison

Once a base context is selected, the algorithm compares each of its objects to the objects of every other context. For each object, it finds the best possible match using a 1:1 matching logic. If multiple objects try to match the same target, only the highest-scoring match is retained. Unmatched objects are still counted but contribute zero to the score.

Rarity

After pairing, each matched parameter is adjusted according to its rarity in the entire dataset. Common materials or functions (e.g., bronze, domestic) carry less weight, while rarer elements (e.g., iron weapons) become more influential.

Group-to-Group Similarity

After object matching, the algorithm computes the average similarity score between the two groups. It also considers parameters proper to the context itself (e.g., the typology of the tomb itself) and includes their contribution to the final score.
Objects that are not matched contribute a score of 0, and thus lower the overall average proportionally.

Component Contribution to Final Score
Object matches (3 out of 4) +0.6
Rare unmatched object −0.3
Unmatched object (no rare value) 0.0
Matching static parameter +0.2
Total Affinity Score ≈ 0.5 (scale 0–1)

Users may customize the process by:

  • Reweighting or excluding specific parameters (e.g., emphasize material over function)
  • Selecting a custom triple of parameters that, when matched, grant a score bonus
  • Adjusting the interpretation focus depending on the research question

1. Data Model

Let \( C = \{c_1, c_2, \dots, c_m\} \) be the set of contexts. Each context \( c_i \) has a set of associated objects \( O_i = \{o_1, o_2, \dots, o_n\} \). Each object \( o \in O_i \) is described by a vector of categorical attributes: \[ o = \big(a_1(o), a_2(o), \dots, a_k(o)\big) \] where each \( a_j(o) \subseteq V_j \) is a (possibly multi-valued) set of values for attribute \( j \). A distinguished triplet \( T = (a_{i_1}, a_{i_2}, a_{i_3}) \) can be selected for bonus scoring.

2. Attribute Weighting and Rarity

Each attribute-value pair \( (a_j, v) \) is assigned a weight: \[ w(a_j, v) = \frac{1}{\sqrt{f(a_j, v)}} \cdot \lambda_j \] where \( f(a_j, v) \) is the global frequency of value \( v \) for attribute \( j \), and \( \lambda_j \) is a user-defined importance weight. This ensures that rare values contribute more to similarity, akin to inverse-document-frequency logic.

3. Object-to-Object Similarity

a) Attribute-Level Similarity

For each attribute \( a_j \), similarity is computed as: \[ \text{sim}_j(o, o') = \frac{\sum_{v \in a_j(o) \cap a_j(o')} w(a_j, v)}{\max(|a_j(o)|, |a_j(o')|)} \] This gives partial credit for overlapping values, normalized by the larger attribute set.

b) Weighted Attribute Sum

Total similarity across all attributes is computed as: \[ \text{sim}_{\text{attr}}(o, o') = \sum_{j=1}^{k} \lambda_j \cdot \text{sim}_j(o, o') \]

c) Triplet Bonus

If the full triplet \( T = (a_{i_1}, a_{i_2}, a_{i_3}) \) has matching values, a bonus is added: \[ \text{bonus}_T(o, o') = \sum_{\substack{ v_1 \in a_{i_1}(o) \cap a_{i_1}(o') \\ v_2 \in a_{i_2}(o) \cap a_{i_2}(o') \\ v_3 \in a_{i_3}(o) \cap a_{i_3}(o') }} \frac{1}{\sqrt{f_T(v_1, v_2, v_3)}} \]

d) Final Object Similarity

The overall similarity between objects is: \[ \text{sim}(o, o') = \frac{\text{sim}_{\text{attr}} + \text{bonus}_T}{k} \]

4. Context-to-Context Matching

Given two contexts \( c_i, c_j \) with object sets \( O_i, O_j \), a greedy 1:1 matching is performed to maximize total similarity. Let \( M = \{(o, o')\} \) be the set of matched pairs (possibly with null matches). The context similarity is: \[ \text{sim}(c_i, c_j) = \frac{1}{|M|} \sum_{(o, o') \in M} \text{sim}(o, o') \]

5. Static Contextual Similarity

If the static types of both contexts match (e.g. same tomb typology), an additional bonus is computed as: \[ \text{sim}_{\text{static}}(c_i, c_j) = \frac{1}{\sqrt{f_{\text{type}}(v)}} \quad \text{if } \text{type}(c_i) = \text{type}(c_j) = v \] otherwise 0.

6. Final Affinity Score

The final scalar score is: \[ \text{affinity}(c_i, c_j) = \text{sim}(c_i, c_j) + \text{sim}_{\text{static}}(c_i, c_j) \]

Future Prospects

The current prototype is designed as a scalable foundation for future extensions. Development plans include:

  • Additional archaeological contexts, expanding the dataset beyond the current tombs.
  • Hierarchical modeling of sub-contexts (e.g., chambers, deposits, burial phases) to reflect the internal complexity of funerary units.
  • Incorporation of new data types where available, including human remains, archaeozoological findings, and botanical evidence.
  • Introduction of spatial analysis tools, enabling map-based visualizations and geographic queries.
  • Refinement and expansion of the affinity algorithm, with better control over weighting and improved treatment of edge cases.
